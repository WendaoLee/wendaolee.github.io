发现组里技术氛围突然浓厚了起来，感觉我们组里无论是无关水的乐子质量还是生活分享的烟火琐屑事都在蒸蒸日上，真是令人感慨啊。

但与此同时，我发现有朋友被豆瓣的文本编辑器折磨😾最后排版因为豆瓣的问题显示得不咋地。所以在这里，我决定跟大家分享一下如何将自己用 markdown 的文本快速转换为可以直接黏贴到豆瓣（网页端）编辑器的文本——因为我是用 markdown 写作的，所以也只能写这个教程~~

## 言简意赅

当你复制内容的时候，复制到剪切板里的内容通常会存在这样的两种形式（可见我之前的 [译文](https://leewendao.otterstack.cn/writings/translations/the-webs-clip-board-and-how-it-stores-data-of-diffent-types)）：

- 以纯文本表示的 `text/plain`
- 以 HTML 表示的 `text/html`，基本上富文本的格式都通过 HTML 进行保留。

虽然豆瓣的文本编辑器很垃圾，但好消息是它至少支持对于 `HTML` 形式的富文本的解析支持。因此，你只需要找到将 markdown 转换为 HTML 格式的解析器即可。

因此，邪门的方法是：由于只要黏贴 HTML 格式的内容便可以让豆瓣的文本编辑器自己解析，因此你可以考虑把文章先发到其他 markdown 支持的网站上，e.g. 稀土掘金，知乎，然后从那边复制黏贴过来~

所以对于很多帖子，我的做法是直接从我自己的博客站那边复制过来~~

同样，你可以去寻找相应的 markdown 转豆瓣的解析器。

网上这类转换的比较多。我平常用的比较多的是这几个：

1. [前辈做的 markdown to something，专门转豆瓣](https://greenzorro.github.io/markdown-to-something/)
2. [陆理的markdown转微信公众号文章](http://md.poycode.cn/)
3. [Md2All](https://md2.sleele.com/)

虽然平日里第二个我用得比较多——因为我写的有些博文会发到我自己的个人公众号上——但第一个效果还算好，能保留链接图片之类的格式。

一般情况下，你只需要把你的 markdown 文本复制黏贴到那里，到时候黏贴过来就行了。

其余的在你发现格式有大问题的时候可以试试。总之，因为网上不同的转换工具它们的样式与转换规则不同，因此黏贴的效果也可能不同。有的时候也许别的转换规则会更适合你的文章。可以多试试。


## 但是，豆瓣确实很垃圾...

一个需要你注意的事情是，如果你的 markdown 里有图片，请挂完整的、互联网可访问到的图片路径而不是相对路径（这个不用解释）。

**但是** ，即时你挂了完整的URL链接，豆瓣依然可能会解析失败~~

![豆瓣解析失败的示例](/img/【Essay】如何使用markdown在豆瓣上写作/1-douban-failed.png)

这种情况只能你手动上传了~这也是为啥有些文章我懒得发豆瓣的原因之一。

并且，豆瓣不支持代码块，所以如果有代码内容，要么只能傻傻地放文字，要么只能截张图传上来。

但总地来说，基本上大部分内容分享够用了。

如果你遇到解析抽风的情况，不该变成标题的内容变成了标题：

![这里的内容本不该是标题](/img/【Essay】如何使用markdown在豆瓣上写作/1-douban-failed.png)

你只需要鼠标光标停留在那一行上，然后在左上方设置格式即可——是的，豆瓣的文本编辑器有点反直觉，它标题之类的格式设置是基于行的，而不像word那样要你用鼠标框选设置格式。

除此之外，在用 markdown 写作时，你需要注意大部分的 markdown 解析器都是基于 [Github Flavor Markdown](https://docs.github.com/zh/contributing/writing-for-github-docs/using-markdown-and-liquid-in-github-docs) 这个标准的，因此如果你的书写用了 Obsidian 之类的 markdown 编辑器自己的方言（e.g. `[[]]`），很容易出现问题。这一点虽然注意。
